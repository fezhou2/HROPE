Hierarchical Rotary Position Embedding (HRoPE)

A symmetric U-Net text encoder-decoder for editable LLMs — sentence-isolated meaning, hierarchical position bands, and exact incremental edits

📄 Feng Zhou — April 2026 All Rights Reserved Unified Technical Specification (v4 → v7 consolidated)
⚡ Scope of this document

This is the consolidated specification covering only the features that survived the v3 → v7 evolution and are present in the v7 reference implementation. Earlier-version-only ideas (the v3 monolithic 50M correction model, the v4/v5 flat-transformer assumption, ALiBi at L1/L2, and the v6 single cross-attention broadcast) have been superseded and are not included.

The retained core: a hierarchical encoder-decoder with three levels (token / sentence / paragraph), disjoint RoPE bands per level, sentence-isolated L0 attention, a symmetric decoder path with additive skip connections at every level, and a four-component cache-stitching toolkit (denominator-tracked ReLU² attention, QK-Norm, parallel residual, micro-correction gates) used selectively where it matters.

1. Problem Statement

Standard transformer LLMs entangle token meaning with document position: every key/value vector depends on its absolute (or relative) place in the full sequence. Editing one sentence in a 100k-token document forces O(N²) attention recomputation across the rest of the document — several seconds of GPU time, unacceptable for iterative editing or long-form agent workflows.

Goal

The representation of any sentence depends only on its own tokens. Inter-sentence relationships are carried by a separate stream of one-token-per-sentence summaries. Editing sentence k recomputes only that sentence at L0, plus a cheap pass over the sentence-summary stream at L1 (and optionally one paragraph row at L2). Document context still reaches the LM head, via a symmetric decoder path that broadcasts contextualized summaries back down through skip connections.

2. The nnU-Net Architectural Principle

nnU-Net (Isensee et al., Nature Methods 2021) processes medical images at a hierarchy of resolutions: a coarse encoder captures whole-organ structure; a fine encoder captures cell-scale texture; skip connections from each encoder level into the matching decoder level let the decoder restore fine-grained detail while still using global context. Different scales answer different questions, and the U-shape lets every scale contribute to the final output.

HRoPE applies the same pattern to text:

tokens ↓ L0_enc ──skip─────────► L0_dec → LM head ↓ pool ↑ unpool + skip L1_enc ──skip─────────► L1_dec ↓ pool ↑ unpool + skip L2_enc ──(bottleneck)─► L2_dec

2.1 Three architectural invariants

  1. Sentence isolation at L0. Token-level attention is masked block-diagonally: token t in sentence s can attend only to other tokens in sentence s. RoPE positions reset to 0 at every sentence boundary. The L0 representation of sentence s is independent of which sentence index it occupies in the document. Verified empirically: the smoke test reports max diff = 0.0 in fp32 under L1/L2 reordering.
  2. Higher-level relations are tokens at upper levels. One summary token per sentence is the unit of information at L1. One summary token per paragraph at L2. Standard transformer machinery (causal attention + RoPE + RMSNorm + GLU) operates uniformly at every level — only the sequence it operates over changes.
  3. Top-down is symmetric. The decoder is a mirror image of the encoder. Each decoder level receives unpool(parent_decoder) + skip_from_matching_encoder and runs its own transformer stack. The LM head reads only from the L0 decoder output, so document context arrives via the unpool path while sentence-local detail is preserved by the skip.

2.2 Why this is the right inductive bias for editable text

  • Locality of meaning. A sentence’s token-level interpretation is largely determined by its own tokens (compositionality at the clause level). Forcing it architecturally costs almost nothing on tasks where the upper-level broadcast can fill in the rest.
  • Locality of edits. Document edits are sentence-bounded with overwhelming probability: human revisions, agent rewrites, retrieval-augmented insertions all replace a span that aligns with a sentence/paragraph boundary >90% of the time.
  • Cache invalidation matches the edit hierarchy. An edit at sentence k invalidates exactly: row k of the L0 cache, then a re-pass of L1 (short — ~5,000 sentences for a 100k-token doc), and optionally one paragraph row at L2.

3. Mathematical Foundations

3.1 RoPE recap (split layout, GPT-NeoX / Llama / Qwen)

For a vector x ∈ ℝd with even d, write x = (x(1), x(2)) with x(1), x(2) ∈ ℝd/2. Define

RoPE(x, p) = (x(1)⊙cos(pθ) − x(2)⊙sin(pθ),    x(1)⊙sin(pθ) + x(2)⊙cos(pθ))

This is the GPT-NeoX / Llama / Qwen layout. Frequencies follow a geometric progression θi = base−2i/d; low-index dimensions rotate fast (fine position differences), high-index dimensions rotate slowly (broad relationships).

3.2 Three properties of rotation that matter

P1 — Composability: R(θ1) R(θ2) = R(θ1 + θ2). Per-block 2D rotations satisfy the angle-addition identities for sin/cos.
P2 — Invertibility: R(θ)−1 = R(−θ) = R(θ)T. Rotations are orthogonal.
P3 — Norm preservation: ‖R(θ) x‖2 = ‖x‖2. The reason QK-Norm composes with RoPE without destroying relative position information.

3.3 Disjoint RoPE bands per hierarchy level

Every transformer block uses pre-norm RMSNorm + RoPE, but each level’s blocks use a different rotary base. The sentence index never enters the L0 representation; it appears as a position only in the L1 sequence of summary tokens. Distinct sequences cannot alias. This is the v6 fix to the v3 superposed-position collision problem.

L0: θ(0)i = 10000−2i/d,   positions = t (token offset within sentence)
L1: θ(1)i = 1000−2i/d,   positions = φs (sentence offset within paragraph)
L2: θ(2)i = 100−2i/d,   positions = πp (paragraph offset within document)

4. Forward Pass — the Whole Architecture in 30 Lines

def forward(self, doc):
    # ─── ENCODE (going up the hierarchy) ───────────────────────────
    x0 = self.tok_emb(doc.token_ids)
    for blk in self.l0_enc:
        x0 = blk(x0, doc.token_in_sent, attn_mask=intra_sent_mask)
    skip_l0 = x0                              # save for L0 decoder

    x1 = self.sent_pool(x0, doc.sent_id, N_s)
    for blk in self.l1_enc:
        x1 = blk(x1, doc.sent_pos_in_para, attn_mask=causal_l1)
    skip_l1 = x1                              # save for L1 decoder

    x2 = self.para_pool(x1, doc.para_id_per_sent, N_p)
    for blk in self.l2_enc:
        x2 = blk(x2, doc.para_pos, attn_mask=causal_l2)

    # ─── DECODE (coming back down with skips) ──────────────────────
    y2 = x2                                   # bottleneck = top of U
    for blk in self.l2_dec:
        y2 = blk(y2, doc.para_pos, attn_mask=causal_l2)

    y1 = unpool(y2, doc.para_id_per_sent) + skip_l1     # SKIP
    for blk in self.l1_dec:
        y1 = blk(y1, doc.sent_pos_in_para, attn_mask=causal_l1)

    y0 = unpool(y1, doc.sent_id) + skip_l0              # SKIP
    for blk in self.l0_dec:
        y0 = blk(y0, doc.token_in_sent, attn_mask=intra_sent_mask)

    return self.lm_head(self.head_norm(y0))

unpool(parent_seq, child_to_parent)[c] = parent_seq[child_to_parent[c]] is a pure indexed gather — zero parameters, one kernel launch. It is the text equivalent of bilinear upsampling in image U-Nets.

5. Components — Per Level

5.1 L0: token-level encoder/decoder (sentence-isolated)

Mask: M[s, q, k] = 1 iff sentences agree (sq = sk) AND k ≤ q (causal within sentence).
Attention: ReLU² with explicit per-row L1 denominator (§6.1).
Norm: RMSNorm pre-norm, plus QK-Norm on Q and K (§6.2).
RoPE: base 10000, positions = intra-sentence token offset t.

Under this mask, the output for token t in sentence s is a function only of {xs,0, …, xs,t}. Verified bit-exactly: scrambling the document’s sentence indices and paragraph positions changes the L0 output by exactly 0.0 in fp32.

The L0 decoder uses the same intra-sentence mask. Its input for token t in sentence s is

y0[s, t] input = skip_l0[s, t] + unpool(L1_dec)[s, t]

Both terms depend only on (a) the tokens of sentence s and (b) one summary vector. Edits to sentence k ≠ s can change L1_dec but cannot reach skip_l0[s].

5.2 Sentence pool (encoder → L1 input)

Per-sentence learnable-query attention pool produces one vector per sentence:

us = ∑t in Ss softmaxt(<qpool, kt>) · vt

The pool is differentiable through L0. It is the only place where intra-sentence content gets compressed for upper layers.

5.3 L1: sentence-level encoder/decoder

Standard pre-norm transformer over the sequence U = (u1, …, uNs). Causal mask. Positions are φs — the sentence’s offset within its paragraph. This makes the L1 sub-sequence corresponding to a paragraph translation-invariant under paragraph reordering, which is a useful inductive bias for retrieval-augmented inputs.

5.4 Paragraph pool + L2 (optional but enabled by default)

Same pattern: pool L1 outputs by para_id_per_sent → one vector per paragraph → transformer over the paragraph sequence with positions πp. The L2 stream is the bottleneck of the U.

5.5 Components removed during the v3–v7 evolution

  • v3 monolithic 50M correction model — eliminated by the four-component toolkit (§6) plus sentence isolation; the decomposable attention is exact at L0 by construction.
  • v4 ALiBi cross-segment bias. Replaced by RoPE on per-level positions; ALiBi at upper levels was directionally incorrect in v3 (B3) and unnecessary once positions live in their own sequences.
  • v6 cross-attention broadcast (broadcast_l1, broadcast_l2). Replaced by the symmetric decoder + additive skip path. Skip is dense, parameter-free, and one operation cheaper than the cross-attention pattern with sparse window mask.

6. The Cache-Stitching Robustness Toolkit (carried over from v4 / v5.1)

HRoPE’s native architecture sidesteps most of the v3 LayerNorm-leakage pathologies by construction (RMSNorm + sentence-isolated attention — see §7). The four-component toolkit below adds the remaining hardening for robust cross-level stitching, especially at L1/L2 where edits do propagate causally.

6.1 Component 1 — Denominator-tracked ReLU² attention

Replace softmax with squared ReLU and expose the per-row L1 denominator:

φ(z) = max(0, z)2,   Zq = ∑k Mqk φ(qTk),   Attn(q) = ∑k Mqk φ(qTk) vk / (Zq + ε)
Theorem — exact decomposability over key segments

For any partition of keys into segments {S1, …, Sm}, define per-segment sufficient statistics

Nj(q) = ∑i∈Sj φ(qTki) vi,   Dj(q) = ∑i∈Sj φ(qTki).

Then Attn(q, K, V) = (∑j Nj) / (∑j Dj + ε). Combination is addition + one scalar division. No approximation. ■

This contrasts with softmax, where the global exp-sum denominator is non-additive across segments. ReLU² preserves the sparsity benefits of ReLU (kills negatives) while keeping a smooth gradient at zero (derivative 2z, continuous), and matches softmax quality to within ~1% in language modeling (Google Primer).

6.2 Component 2 — QK-Norm (logit bounding)

L2-normalize Q and K before the dot product, then scale by a learned per-head temperature:

q̂ = q / ‖q‖2,   k̂ = k / ‖k‖2,   score = τ · q̂Tk̂
Theorem — QK-Norm preserves RoPE’s relative-position encoding

Let qraw, kraw be the pre-RoPE projections and q = R(pqθ) qraw, k = R(pkθ) kraw. By P3, ‖q‖ = ‖qraw‖ and ‖k‖ = ‖kraw‖ (position-independent). Therefore

q̂Tk̂ = qrawT R((pk−pq)θ) kraw  /  (‖qraw‖ ‖kraw‖).

The relative-position term R(Δp · θ) is fully preserved; only content-based norms divide. ■

QK-Norm is used at every level. Production-validated in Meta’s Chameleon and Google’s ViT-22B.

6.3 Component 3 — Parallel residual blocks (selective per level)

Sequential residual feeds the FFN with the post-attention state, so attention error compounds through the FFN at every layer. Parallel residual (PaLM, Gemma) decouples them:

# Sequential                              # Parallel
x = x + Attn(RMSNorm(x))                  h = RMSNorm(x)
x = x + FFN(RMSNorm(x))                   x = x + α·Attn(h) + β·FFN(h)

where α, β are learned per-block scalars (init 0.5 each, ~2 extra parameters per block). Per-layer error amplification drops from roughly (1+C)(1+LF) (sequential) to 1 + αC + βLF (parallel). With α ≈ β ≈ 0.3, C ≈ 1, LF ≈ 1: 1.6 vs 4.0 per layer.

LevelDefaultReason
L0 encoder/decoderOFFSentence-isolated; nothing to compound across the L0 stack.
L1 encoder/decoderONConsumes cross-sentence context; bounds error if a downstream call uses approximate stitching.
L2 encoder/decoderONSame reasoning as L1 at paragraph granularity.

6.4 Component 4 — Cached segment statistics & micro-correction gates (opt-in)

For approximate cross-segment stitching at L1 (the v8 incremental-L1 path), cache lightweight per-segment statistics alongside each sentence’s skip representation:

μs(l,h) = meani∈S(ki(l,h)),   logσs(l,h) = log stdi∈S(ki(l,h)).

Storage: for a 32-layer, 32-head, dhead=128 model: 2 vectors × 32 layers × 32 heads × 128 dims × 2 bytes (bf16) = ~0.5 MB per segment. Negligible vs the segment’s KV cache (50–200 MB).

At stitch time, a small gate consumes pairwise (Δμ, Δlogσ) features and produces a per-head sigmoid that scales the attention output. Two variants:

Scalar gate (default)Vector gate
OutputOne scalar per head per layerOne vector ∈ ℝdhead per head per layer
Total params (32 layers)~260K~33M
InitZero weights ⇒ sigmoid(0) = 0.5 (identity-like at init, multiplicative bias absorbed by Wo)Same
When to enablev8 approximate-L1 pathIf accuracy gap with scalar gate exceeds ~0.5%

Default in v7: gate is wired and zero-initialised (identity at init), but disabled, because the v7 editor does an exact L1 re-pass and there is no residual drift to correct. The gate is the parameter machinery a future v8 incremental-L1 editor needs.

7. Why the v3 LayerNorm-Leakage Scenario Doesn’t Apply Natively

The v3/v4 LayerNorm-leakage concern was framed for Path 1 retrofit: take an existing Llama-3 / Qwen model trained with full softmax attention and try to swap in cache stitching at inference. In that setting, hidden-state distributions drift away from what the model expects, and a correction model (or micro-gates) is needed to compensate.

HRoPE’s native path (Path 2) sidesteps this at the L0 layer by construction:

  1. RMSNorm, not LayerNorm. RMSNorm has no mean-subtraction step; it normalises each per-token vector by its own L2 magnitude only. There is no cross-token statistic, so an edit at sentence k cannot shift sentence j’s normalised representation through a shared mean.
  2. Sentence isolation enforced by mask. The L0 attention mask is block-diagonal at sentence boundaries; sentence k’s tokens never enter sentence j’s key/query/value computation through any path.
  3. Train/inference parity. The model is trained with this exact masking. The hidden states the L0 encoder produces under stitched inputs are the same it would produce under a fresh full-document forward, because the only thing the L0 encoder ever sees during training is sentence-isolated context.

Where stitching effects do appear — L1 self-attention (causal across sentences) and the L0 decoder broadcast input — the v7 editor handles them with an exact re-pass at L1/L2 and trained robustness in Stage 3 of the curriculum. The four-component toolkit (§6) is opt-in for downstream approximate-stitching paths.

8. Incremental Edit Cost

Edit a single sentence k on a 100k-token document (~5,000 sentences, ~250 paragraphs):

1L0_enc on sentence k only (intra-sentence causal)O(Lk2) — tens of µs
2Splice skip_l0[k] into cacheO(Lk)
3Re-pool sentence k; re-run L1_enc over the spliced sentence streamO(Ns · d · n_l1_enc) ≈ few ms
4Re-pool affected paragraph; re-run L2_encO(Np · d) ≈ sub-ms

The L0_dec for non-edited sentences k+1 … Ns−1 is technically also affected (their L1_dec broadcast vector changed because L1 attention is causal). For the standard editor flow — where you only need correct logits at the cursor — you skip those. For a full re-decode of the document tail, add ~5−10 ms more.

9. Sentence-Level Fuzzy Cache (SimHash with character n-grams)

To recognise “edits” that are textually near-identical to a previously cached sentence (e.g. agent re-typing the same line, retrieval re-fetching a near-duplicate), HRoPE uses SimHash over character n-grams (n = 3 default) rather than whitespace tokens. This is the v6 fix to v3’s whitespace-only featurisation, which failed on Chinese / Japanese / Thai / any language without whitespace.

def simhash(sentence, hash_bits=64, n=3):
    grams = [sentence[i:i+n] for i in range(len(sentence) - n + 1)]
    v = [0] * hash_bits
    for g in grams:
        h = int.from_bytes(hashlib.md5(g.encode()).digest(), 'big')
        for i in range(hash_bits):
            v[i] += 1 if (h >> i) & 1 else -1
    return sum((1 << i) for i in range(hash_bits) if v[i] > 0)

# Cache lookup: query for entries with Hamming distance ≤ 3 bits.
# If hit -> reuse cached skip_l0; no re-encode needed.

11.3 Data preparation

  1. Sentence segmentation: spaCy en_core_web_sm (English) + jieba (zh) + SudachiPy (ja). Per-language pipeline; never split inside code blocks or LaTeX.
  2. Paragraph detection: blank-line boundaries in text, header-level boundaries in markdown.
  3. Cap sentence length at 256 tokens; longer sentences hard-split at the comma boundary closest to the cap.
  4. Cap paragraphs at 64 sentences; longer paragraphs split at the sentence boundary closest to the cap.
  5. Documents up to 64 paragraphs (~32k−65k tokens) used unmodified; longer ones sliced into overlapping windows of 32 paragraphs with 4-paragraph overlap.

11.4 Stage 3 perturbations

  • Probability 0.2: permute the sentence-summary stream within each paragraph.
  • Probability 0.1: drop one paragraph entirely (zero out its summary).
  • Probability 0.1: replace one sentence’s tokens with a random sentence from a different document and recompute its L0 embedding (simulates retrieval-augmented insertion).

Stage 3 is the cheapest stage (~7% of total) but it is what makes the final model robust to the inference-time edit workload.

Honest caveats
  • The 0% accuracy drop relative to a full-recompute upper bound is a property of the architecture, not of the trained model’s quality. A v7 model will measure differently against a frontier dense-attention model on raw next-token loss because the inductive bias (sentence isolation at L0) costs some capacity. The relevant benchmark is “v7 native vs an equivalently-sized dense transformer trained on the same tokens.” That comparison is the open empirical question (see §14).
  • The ~5 s standard baseline assumes no prefix caching. Modern serving stacks (vLLM, SGLang, TGI) with prefix caching already handle append-only edits efficiently. HRoPE’s differentiation is on mid-document edits (insert/delete/replace in the middle), where prefix caching cannot help.

14. Feasibility & Risks

14.1 What is unambiguously sound

  • Architecture compiles, runs, and respects its invariants. The reference produces L0 invariance ≡ 0.0 empirically.
  • RoPE math is exact. Composability and norm preservation are textbook facts; the only thing v3 got wrong was the layout (interleaved vs split), which is fixed.
  • Edit cost is dominated by L1 self-attention (5−15 ms for 5k sentences). Straightforward arithmetic, matches measurements of similarly-sized transformers.
  • Errata fixes are mechanical. All ten v3 bugs (B1–B10) are reproducible with unambiguous fixes (§10).

14.2 What is unproven (and how to prove it)

  • Quality on next-token loss vs equivalently sized dense transformer. The sentence-isolation prior is a constraint; it could cost >1 point of perplexity at scale. De-risk: 350M ablation v7 vs 350M dense on same 30B tokens (~$5K, ~3 days, 8×H100). If within ~5% of dense PPL, scale up.
  • Sentence-summary capacity. One vector per sentence may not be enough for long or technical sentences. De-risk: pool to k summary tokens per sentence (k = 1, 2, 4) and pick the smallest k that closes the gap. Reference’s AttentionPool can be parameterised for this with ~5 lines of change.
  • L1 self-attention bottleneck on incremental edits. 5−15 ms is fine for human-in-the-loop editing but could matter for high-QPS agent workloads. De-risk: approximate L1 stitching using the cached statistics + micro-correction gates (the v8 path; the parameter machinery already ships in v7.1).
  • Stage 0 actually learns useful sentence representations. Forcing L0 to predict next-token from intra-sentence context only is a strong constraint. De-risk: add an auxiliary span-corruption loss at L0 to sidestep the “first token has no context” degeneracy.

14.4 Verdict

Feasibility: HIGH for the architecture, MEDIUM for the quality bet.

The architecture is buildable today, the math is correct, and the reference runs. The unknown is whether the inductive bias costs more in language-model quality than it gains in edit efficiency. A 350M-parameter ablation against a dense baseline is the single experiment that decides this; it costs ~$5K and ~3 days. Do that experiment first, before committing to the 1.5B run.
