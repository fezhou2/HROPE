Hierarchical Rotary Position Embedding (HRoPE)

Incremental inference for LLMs via sentence-level caching, re-rotatable KV, and decomposable normalized attention

1. Problem Statement

Standard LLMs recompute the entire KV cache when any token changes. For a 100k-token document, editing one word triggers O(N²) recomputation — several seconds of GPU time. This makes iterative editing prohibitively expensive and prevents true incremental compilation for LLMs.

Our Goal: Edit a sentence, recompute only that sentence. Everything else is served from a precomputed, position-agnostic cache.

💡 Analogy — nnU-Net for Medical Imaging nnU-Net processes images at multiple resolutions (organ → tissue → cell). HRoPE processes text at multiple levels (paragraph → sentence → token). This hierarchical structure enables efficient, incremental updates and multi-scale pattern recognition.

2. Core Mathematical Insight: The Composability of Rotation

RoPE (Rotary Position Embedding) applies a rotation to key vectors based on absolute position. For a vector x ∈ ℝd with even d:

RoPE(x, p) = R(pθ) · x

where:

  • p — absolute position
  • θi = 10000−2i/d for i = 0, 1, …, d/2 − 1
  • R(pθ) — block-diagonal rotation matrix

Rotation matrices are composable and invertible:

R(θ1)R(θ2) = R(θ1 + θ2)    R(θ)−1 = R(−θ)

If we store the unrotated key vectors Kbase, we can rotate them to any target position later:

Kcached = R(poldθ) · Kbase
Knew = R(pnewθ) · Kbase = R((pnew − pold)θ) · Kcached

This means a sentence’s KV cache is position-neutral. It works anywhere in any document. No approximation — the math is exact.

4. The Four-Component Architecture NEW in v4

The v3 design used “ReLU + renormalization” (undefined) plus a 50M-parameter correction model to patch LayerNorm leakage. This v4 design replaces both with a mathematically rigorous architecture where stitching is provably exact and LayerNorm leakage is eliminated by construction.

4.1 Component 1: Denominator-Tracked ReLU² Attention

Why ReLU² instead of plain ReLU or softmax?

Definition

For a query q attending to keys K = {k1, …, kn} with values V = {v1, …, vn}:

Attn(q, K, V) = ∑i φ(qTki) · vi / (∑i φ(qTki) + ε)

where φ(z) = max(0, z)² (squared ReLU) and ε = 10−6.

Theorem: Exact Decomposability

🔎 Proof of Stitching Exactness

For any partition of keys into segments {S1, …, Sm}, define per-segment sufficient statistics:

Nj(q) = ∑i∈Sj φ(qTki) · vi ∈ ℝdv    Dj(q) = ∑i∈Sj φ(qTki) ∈ ℝ

Then:

Attn(q, K, V) = ∑j Nj(q)  /  (∑j Dj(q) + ε)

Each (Nj, Dj) depends only on q and segment Sj’s cached (K, V). Combination is addition + one scalar division. No approximation. ▮

Composition with RoPE: Keys are re-rotated before the dot product: kiglobal = R(piglobalθ) · kibase. Since re-rotation is per-key and happens before score computation, the decomposition is unaffected.

Contrast with softmax: Softmax requires exp(qTki) / ∑j exp(qTkj). The exponential denominator is non-additive across segments (requires the FlashAttention online-softmax trick with max correction). ReLU² avoids this entirely because both numerator and denominator are simple sums.

Contrast with unnormalized ReLU (v3 spec): Unnormalized ReLU output grows as O(n) with sequence length, causing training instability for long contexts. The denominator tracking adds normalization without breaking decomposability.

4.2 Component 2: QK-Norm (Logit Bounding) NEW

Normalize queries and keys to unit vectors before computing attention scores:

q̂ = q / ‖q‖2    k̂ = k / ‖k‖2    score = τ · q̂Tk̂

where τ is a learned per-head temperature scalar (initialized to √dhead).

Why this matters for stitching:

  • All attention logits are bounded to [−τ, +τ] regardless of hidden state magnitude
  • When a chunk is encoded independently (missing cross-chunk context), its hidden states may have different norms than in joint encoding. Without QK-Norm, this magnitude difference directly corrupts attention scores. With QK-Norm, only direction matters — direction is far more robust to missing context
  • The temperature τ is a fixed learned parameter, not data-dependent, so it introduces no stitching error
📚 Literature QK-Norm is used in production models (Chameleon by Meta, ViT-22B by Google). It stabilizes training and reduces attention entropy variance — both critical for cache stitching.

4.3 Component 3: Parallel Residual Blocks (Error Isolation) NEW

In a standard sequential transformer block, each layer computes:

x = x + Attn(RMSNorm(x))
x = x + FFN(RMSNorm(x))       # FFN input depends on attention output

The FFN input depends on the attention output. When attention is wrong (missing cross-segment context), the error propagates through FFN, then into the next layer’s attention, compounding exponentially through L layers.

Parallel residual (used in PaLM, Gemma) decouples them:

h = RMSNorm(x)
x = x + α · Attn(h) + β · FFN(h)    # both computed from SAME input

where α, β are learned per-layer scalars.

PropertySequential ResidualParallel Residual
FFN depends on attention output?YesNo
Context-independent fraction of layer output 0% (everything flows through attention) ~50–70% (the FFN branch)
Error propagation across L layers O(1L) = O(1) — full propagation O(αL) — exponential decay (α ≈ 0.3)
Hidden state divergence at layer 32 Unconstrained α32 ≈ 10−17 (negligible)

Why this replaces the 50M correction model: With parallel residual, independent encoding produces nearly identical hidden states to joint encoding at every layer. The FFN branch — which constitutes the majority of each layer’s contribution — is completely unaffected by the attention difference.

4.4 Component 4: Cached Segment Statistics & Micro-Correction Gates NEW

For any small remaining divergence (which Components 1–3 have already minimized), store lightweight statistics alongside each segment’s KV cache:

Per segment S, per layer l, per head h, cache two extra vectors:

μs(l,h) = meani∈S(ki(l,h)) ∈ ℝdhead    σs(l,h) = stdi∈S(ki(l,h)) ∈ ℝdhead

Storage overhead: For a 32-layer, 32-head, dhead = 128 model:
2 vectors × 32 layers × 32 heads × 128 dims × 2 bytes (bf16) = ~0.5 MB per segment. Negligible.

At stitching time, compute a per-layer correction gate from cross-segment statistics:

def micro_correction(attn_out, seg_stats_list, W_gate, b_gate):
    """Per-layer gate from segment statistics. ~3*d_head^2 params per layer."""
    # seg_stats_list: list of (mu, sigma) per segment
    # Pairwise difference features between adjacent segments
    delta = concat([mu_i - mu_j, sigma_i / (sigma_j + eps)
                    for i, j in adjacent_pairs(seg_stats_list)])
    gate = sigmoid(W_gate @ delta + b_gate)  # (n_heads, d_head)
    return attn_out * gate

4.5 Why This Eliminates LayerNorm Leakage

The v3 spec identified “LayerNorm leakage” as the drift in normalization statistics when stitching independently encoded segments. The v4 architecture eliminates this at four levels:

LevelMechanismEffect
Normalization RMSNorm (no mean subtraction) in pre-norm position K, V normalized before caching; no post-cache normalization needed
Logit bounding QK-Norm constrains scores to [−τ, +τ] Hidden state magnitude changes from missing context don’t affect attention scores
Error isolation Parallel residual decouples FFN from attention ~70% of each layer’s update is context-independent; error decays as αL
Residual correction Cached segment statistics + per-layer gate Remaining divergence corrected with ~1.6M total params
🎯 Net result The 50M-parameter correction model from v3 is eliminated. The architecture is self-correcting by design. Stitching is mathematically exact at the attention level, and hidden-state divergence is bounded to negligible levels by parallel residual.

5. Complete HRoPE Transformer Layer

class HRoPEBlock(nn.Module):
    def __init__(self, d_model, n_heads, d_head, d_ff):
        self.norm = RMSNorm(d_model)             # Pre-norm (not LayerNorm)
        self.W_q = nn.Linear(d_model, n_heads * d_head, bias=False)
        self.W_k = nn.Linear(d_model, n_heads * d_head, bias=False)
        self.W_v = nn.Linear(d_model, n_heads * d_head, bias=False)
        self.W_o = nn.Linear(n_heads * d_head, d_model, bias=False)
        self.tau  = nn.Parameter(torch.ones(n_heads) * math.sqrt(d_head))
        self.ffn  = SwiGLU(d_model, d_ff)
        self.alpha = nn.Parameter(torch.tensor(0.5))  # attention weight
        self.beta  = nn.Parameter(torch.tensor(0.5))  # FFN weight
        # Micro-correction gate (used only at stitch time)
        self.W_gate = nn.Parameter(torch.zeros(n_heads, d_head, 2 * d_head))
        self.b_gate = nn.Parameter(torch.zeros(n_heads, d_head))

    def forward(self, x, rope_cos, rope_sin, causal_mask, segment_stats=None):
        B, S, _ = x.shape
        h = self.norm(x)

        Q = self.W_q(h).view(B, S, H, D).transpose(1, 2)
        K = self.W_k(h).view(B, S, H, D).transpose(1, 2)
        V = self.W_v(h).view(B, S, H, D).transpose(1, 2)

        # Apply RoPE (hierarchical positions)
        Q = apply_rope(Q, rope_cos, rope_sin)
        K = apply_rope(K, rope_cos, rope_sin)

        # QK-Norm: project onto unit sphere, scale by learned temperature
        Q = Q / (Q.norm(dim=-1, keepdim=True) + 1e-6)
        K = K / (K.norm(dim=-1, keepdim=True) + 1e-6)

        # ReLU-squared attention with denominator tracking
        scores = torch.einsum('bhsd,bhtd->bhst', Q, K) * self.tau[:, None, None]
        scores = scores + causal_mask               # apply causal + ALiBi bias
        weights = scores.clamp(min=0).square()       # phi(z) = ReLU(z)^2

        denom = weights.sum(dim=-1, keepdim=True) + 1e-6
        attn_out = (weights @ V) / denom             # normalized weighted sum

        # Micro-correction (only when stitching cached segments)
        if segment_stats is not None:
            gate = torch.sigmoid(
                torch.einsum('bhd,hde->bhe', segment_stats, self.W_gate)
                + self.b_gate
            ).unsqueeze(2)
            attn_out = attn_out * gate

        attn_out = self.W_o(attn_out.transpose(1, 2).reshape(B, S, -1))
        ffn_out  = self.ffn(h)                       # parallel: same input as attention

        return x + self.alpha * attn_out + self.beta * ffn_out

6. KV Cache Stitching Protocol

At inference time, stitching m segments proceeds as:

For each segment s = 1..m:
    1. Load cached (K_base_s, V_s, mu_s, sigma_s) from storage
    2. Re-rotate keys: K_s = R(p_s_global * theta) * K_base_s   # exact
    3. QK-normalize:   K_s = K_s / ||K_s||                       # per-key

For each query token q:
    4. Project & normalize: Q = QK_Norm(RoPE(W_q(RMSNorm(x_q))))
    5. For each segment s:
         N_s = sum_i phi(tau * Q . K_s[i]) * V_s[i]   # numerator partial sum
         D_s = sum_i phi(tau * Q . K_s[i])             # denominator partial sum
    6. Combine exactly: attn_out = (sum_s N_s) / (sum_s D_s + eps)
    7. Apply micro-correction gate from segment statistics
    8. Parallel residual: x = x + alpha * attn_out + beta * FFN(RMSNorm(x))
✅ Exactness guarantee Step 6 produces exactly the same output as computing attention over the full concatenated sequence. No approximation at any point. The only non-exact component is the micro-correction gate (Step 7), which compensates for hidden-state divergence that is already bounded to O(αL) by the parallel residual.

7. Mathematical Formulation: Two-Level Hierarchy

For a token at sentence index i and token offset j:

Θ(i, j) = αsent · i + αtoken · j

where each α is a per-dimension frequency vector ∈ ℝd/2:

  • αsent,k = λ · 10000−2k/d — sentence-level (slow rotation, large λ ≈ 100–500)
  • αtoken,k = 10000−2k/d — token-level (standard RoPE frequencies)

The sentence gap λ is a learned scalar initialized to the average sentence length in the training corpus.

The attention score between tokens at (i, j) and (i′, j′) depends on:

ΔΘ = αsent(i − i′) + αtoken(j − j′)

7.1 Variable-Length Sentences (No Fixed Limits)

Document = [S1, S2, …, SN],   Si = [ti,1, …, ti,Li]
  • N — number of sentences (variable)
  • Li — number of tokens in sentence i (variable per sentence)

7.2 ALiBi for Cross-Segment Bias (Global Order)

To encode global order across segments, we integrate ALiBi (Attention with Linear Biases) based on segment index difference:

AttentionGlobal(Q, K) = τ · Q̂TK̂ − m · |segi − segj|

where m is a per-head slope (geometric sequence: mh = 2−8h/H).

📚 Literature Reference The 2025 BiPE (Bilevel Positional Encoding) paper validates this approach, achieving 97% accuracy on arithmetic tasks vs. <70% baseline. Our extension adds QK-Norm and ReLU² decomposability.

8. The Re-Rotation Kernel (Triton) BUG FIXED

@triton.jit
def rerotate_kv_kernel(K_base_ptr, K_out_ptr, start_pos, seq_len, d_head):
    token_idx = tl.program_id(0)
    dim_idx   = tl.program_id(1)

    theta = 10000.0 ** (-2.0 * dim_idx / d_head)
    angle = (start_pos + token_idx) * theta     # v3 bug: was `start_pos * theta`

    cos_theta = tl.cos(angle)
    sin_theta = tl.sin(angle)

    offset = token_idx * d_head + dim_idx * 2
    x = tl.load(K_base_ptr + offset)
    y = tl.load(K_base_ptr + offset + 1)

    tl.store(K_out_ptr + offset,     x * cos_theta - y * sin_theta)
    tl.store(K_out_ptr + offset + 1, x * sin_theta + y * cos_theta)
⚠️ v3 → v4 bug fix The v3 kernel used angle = start_pos * theta, which applied the same rotation angle to every token in the chunk. The correct formula is (start_pos + token_idx) * theta, giving each token its own global position. The PyTorch reference implementation in the repo was already correct.

9. LSH-Based Fuzzy Cache (SimHash)

def simhash(sentence, hash_bits=64):
    features = sentence.lower().split()
    v = [0] * hash_bits
    for f in features:
        h = hashlib.md5(f.encode()).digest()
        h_int = int.from_bytes(h, 'big')
        for i in range(hash_bits):
            if (h_int >> i) & 1:
                v[i] += 1
            else:
                v[i] -= 1
    signature = 0
    for i in range(hash_bits):
        if v[i] > 0:
            signature |= (1 << i)
    return signature

# "The dog sat" -> 0x7f3a2b1d...  (Hamming distance = 1-2 bits)

11.1 Fine-Tuning Strategy

  1. Phase 1 (steps 1–2000): Replace softmax with ReLU², add QK-Norm. Train with full-context encoding (standard causal attention) to adapt the model to the new attention function.
  2. Phase 2 (steps 2001–4000): Convert to parallel residual. Introduce segment-independent encoding: randomly split documents into segments, encode independently, stitch KV caches. Train on stitched outputs.
  3. Phase 3 (steps 4001–6000): Enable micro-correction gates. Train with mixed stitched/full-context examples. Gradually increase sequence length (2K → 8K → 32K).
