1. Problem Statement

Standard LLMs recompute the entire KV cache when any token changes. For a 100k-token document, editing one word triggers O(N²) recomputation — several seconds of GPU time. This makes iterative editing prohibitively expensive and prevents true incremental compilation for LLMs.

Our Goal: Edit a sentence, recompute only that sentence. Everything else is served from a precomputed, position-agnostic cache.

💡 Analogy — nnU-Net for Medical Imaging nnU-Net processes images at multiple resolutions (organ → tissue → cell). HRoPE processes text at multiple levels (paragraph → sentence → token). This hierarchical structure enables efficient, incremental updates and multi-scale pattern recognition.

2. Core Mathematical Insight: The Composability of Rotation

2.1 RoPE Recap

RoPE (Rotary Position Embedding) encodes position by rotating each key and query vector in paired dimensions. For a vector x ∈ ℝd with even d, RoPE groups the dimensions into d/2 pairs and applies a 2D rotation to each pair:

RoPE(x, p) = R(pθ) · x

where R(pθ) is a block-diagonal matrix of d/2 rotation blocks:

R(pθ) = diag&bigl(R2×2(pθ0),  R2×2(pθ1),  …,  R2×2(pθd/2−1)&bigr;)

Each 2×2 block rotates one pair of dimensions by angle p·θi:

R2×2(α) = [ cos(α)   −sin(α)
      sin(α)     cos(α) ]

The frequencies follow a geometric progression:

θi = 10000−2i/d   for   i = 0, 1, …, d/2 − 1

Low-index dimensions rotate fast (capturing fine position differences); high-index dimensions rotate slowly (capturing broad positional relationships).

2.2 Why Rotation Enables Exact Re-Positioning

The key algebraic properties of rotation matrices:

Property 1 — Composability (group structure)
R(θ1) · R(θ2) = R(θ1 + θ2)

Rotating by θ1 then θ2 equals rotating by their sum. This holds per 2×2 block because cos/sin obey the angle-addition identities: cos(a+b) = cos(a)cos(b) − sin(a)sin(b) and sin(a+b) = sin(a)cos(b) + cos(a)sin(b).

Property 2 — Invertibility
R(θ)−1 = R(−θ) = R(θ)T

Every rotation can be undone by rotating in the opposite direction. Equivalently, rotation matrices are orthogonal: RTR = I.

Property 3 — Norm Preservation (critical for QK-Norm compatibility)
‖R(θ) · x‖2 = ‖x‖2   for all θ, x

Rotation does not change the length of any vector. This is because R is orthogonal: ‖Rx‖² = (Rx)T(Rx) = xTRTRx = xTIx = ‖x‖².

This property is why QK-Norm (L2 normalization) does not interfere with RoPE — see Section 4.2.

2.3 The Re-Rotation Formula

If we store the unrotated key vector Kbase = Wk · h (hidden state projected to key space, before any positional encoding), we can position it anywhere:

Step 1: Encode at local position pold
Kcached = R(poldθ) · Kbase
Step 2: Move from pold to pnew
Knew = R(pnewθ) · Kbase

Substituting Kbase = R(−poldθ) · Kcached (by invertibility, Property 2) and applying composability (Property 1):

Knew = R(pnewθ) · R(−poldθ) · Kcached = R((pnew − pold)θ) · Kcached

A single rotation by the position difference moves a cached key to its new position. No need to actually store Kbase; storing Kcached at any chosen reference position pold is equivalent.

The derivation uses composability (Property 1) and invertibility (Property 2). This is exact — no approximation. A sentence’s KV cache is position-neutral and can be slotted into any location in any document by a single rotation.

4. The Four-Component Architecture

Previous versions used “ReLU + renormalization” (v3, undefined) or ReLU + 50M correction model (v3). This architecture replaces all of that with four interlocking components where stitching is provably exact at the attention level, and hidden-state divergence is bounded to negligible levels through the layers.

4.1 Component 1: Denominator-Tracked ReLU² Attention

The problem with softmax stitching

Standard softmax attention for a query q attending to n keys:

Attnsoftmax(q, K, V) = ∑i=1n ( eqTki / ∑j=1n eqTkj ) vi

The denominator ZA∪B = ∑j∈A∪B exp(qTkj) is a global sum over all keys. The numerator is linear and decomposes cleanly, ∑i∈A∪B eqTki vi = ∑i∈A eqTki vi + ∑i∈B eqTki vi, and so does the denominator (ZA∪B = ZA + ZB). But if each segment is normalized locally before being added together,

(broken) ∑i∈A (eqTki / ZA) vi  +  ∑i∈B (eqTki / ZB) vi  ≠  ∑i∈A∪B (eqTki / ZA∪B) vi

because each per-key weight wi = eqTki ends up divided by ZA or ZB rather than by ZA + ZB. The fix — used by FlashAttention’s online softmax — is to track ZA, ZB and re-weight at merge time, but that requires running attention at merge time, defeating the goal of pre-computed segment outputs. This is the fundamental barrier to exact softmax stitching from cached, already-normalized segment outputs.

ReLU² attention: the decomposable alternative

Replace softmax with squared ReLU:

Definition. Attn(q, K, V) = ∑i φ(qTki) · vi  /  (∑i φ(qTki) + ε)

where φ(z) = max(0, z)² and ε = 10−6.

Proof of Exact Decomposability

Theorem (Stitching Exactness)

For any partition of keys into segments {S1, …, Sm}, the normalized ReLU² attention output can be reconstructed exactly from per-segment sufficient statistics.

Proof. Define per-segment quantities:

Nj(q) := ∑i∈Sj φ(qTki) · vi  ∈ ℝdv   (numerator partial sum)
Dj(q) := ∑i∈Sj φ(qTki)  ∈ ℝ      (denominator partial sum)

Since φ is applied elementwise to each score qTki independently, summing over Sj involves only the keys in that segment. Now observe:

∑i∈A∪B φ(qTki) vi = ∑i∈A φ(qTki) vi + ∑i∈B φ(qTki) vi = NA(q) + NB(q)
∑i∈A∪B φ(qTki) = DA(q) + DB(q)

Therefore:

Attn(q, KA∪B, VA∪B) = (NA + NB) / (DA + DB + ε)

Generalizing to m segments:

Attn(q, K, V) = ∑j=1m Nj(q)  /  (∑j=1m Dj(q) + ε)

Each (Nj, Dj) depends only on q and segment Sj’s cached (K, V). Combination is addition of vectors/scalars plus one division. No approximation. ■

⚠️ Important Qualification (from v5 review)

The decomposition above is exact given the same query q. In practice, the query q at layer l is computed from the hidden state at layer l−1, which itself depends on earlier attention results. When segments are encoded independently, these hidden states diverge slightly from joint encoding.

This divergence accumulates with depth at a per-layer factor ρ = 1 + αC + βLF (Section 4.3), which is substantially smaller than the corresponding factor for sequential residual blocks. The micro-correction gate (Section 4.4) absorbs the residual gap. The combined claim is bounded—not zero—hidden-state divergence; empirical validation is required.

Why this works with RoPE re-rotation

At stitching time, each cached key is re-rotated to its global position before computing scores:

kiglobal = R(piglobalθ) · kibase

The score φ(qTkiglobal) is computed per-key after re-rotation. Since re-rotation is per-key and happens before the dot product, the decomposition is unaffected. Each segment’s (Nj, Dj) uses its own re-rotated keys.

4.2 Component 2: QK-Norm (Logit Bounding)

Normalize queries and keys to unit vectors before computing attention scores:

q̂ = q / ‖q‖2    k̂ = k / ‖k‖2    score = τ · q̂Tk̂

where τ is a learned per-head temperature scalar (initialized to √dhead).

Proof: QK-Norm preserves RoPE’s relative position encoding

Theorem (QK-Norm ⊥ RoPE compatibility)

L2 normalization of Q and K after RoPE does not destroy the relative position information encoded by RoPE.

Proof. Let qraw = Wq hq and kraw = Wk hk be the pre-RoPE projections. After RoPE at positions pq and pk:

q = R(pqθ) · qraw    k = R(pkθ) · kraw

By Property 3 (norm preservation), rotation does not change vector norms:

‖q‖2 = ‖R(pqθ) · qraw‖2 = ‖qraw‖2  (position-independent!)

Therefore, the L2-normalized dot product is:

q̂Tk̂ = (q / ‖q‖)T (k / ‖k‖) = qTk / (‖qraw‖ · ‖kraw‖)

Now expand qTk using the rotation:

qTk = qrawT R(pqθ)T R(pkθ) kraw = qrawT R((pk − pq)θ) kraw

Substituting back:

q̂Tk̂ = qrawT R(Δp · θ) kraw  /  (‖qraw‖ · ‖kraw‖)

The relative position term R(Δp · θ) is fully preserved. L2 normalization divides by ‖qraw‖ and ‖kraw‖, which are position-independent quantities (they depend on content hq, hk but not on positions pq, pk). The relative position encoding is intact. ■

⚠️ Note on v5’s RMSNorm alternative

The v5 revision replaced QK-Norm with RMSNorm, claiming L2 normalization “destroys RoPE’s relative distance property.” This claim is incorrect, as proven above. Moreover, the gain-less form RMSNorm(x) = x/RMS(x) = x/(‖x‖/√d) = √d · x/‖x‖ is mathematically equivalent to L2 normalization scaled by √d, with the scale factor absorbed by the learned temperature τ. Caveat: the standard RMSNorm in Llama / HF (with a learned per-dimension gain γ) applies an element-wise rescale γ ⊙ (x/RMS(x)) that is not norm-preserving, and would break the proof. We therefore use gain-less L2 normalization for QK-Norm specifically; the pre-attention RMSNorm on hidden states retains its γ.

Why QK-Norm helps stitching

  • Bounded logits: All attention scores lie in [−τ, +τ], regardless of hidden state magnitude. When a chunk is encoded independently (missing cross-chunk context), its hidden states may have different norms. Without QK-Norm, this magnitude difference directly corrupts attention scores. With QK-Norm, only the direction matters — direction is far more robust to missing context.
  • Temperature is learned, not data-dependent: τ is a fixed parameter per head, not computed from the input. It introduces zero stitching error.
  • Production-validated: QK-Norm is used in Meta’s Chameleon and Google’s ViT-22B. It stabilizes training and reduces attention entropy variance — both critical for cache stitching.

4.3 Component 3: Parallel Residual Blocks (Error Isolation)

The error propagation problem

In a standard sequential transformer block:

x = x + Attn(RMSNorm(x))           # attention output
x = x + FFN(RMSNorm(x))            # FFN input DEPENDS on attention output

The FFN input is the attention output. When attention is wrong (missing cross-segment context), the error flows through FFN, altering the hidden state, which feeds into the next layer’s attention, which feeds into the next layer’s FFN, and so on. Over L layers, errors compound.

Parallel residual: decoupling FFN from attention

The parallel form (used in PaLM-540B and Gemma) computes both from the same input:

h = RMSNorm(x)
x = x + α · Attn(h) + β · FFN(h)    # both computed from SAME normalized input

where α, β are learned per-layer scalars (initialized to 0.5 each).

Error propagation analysis

Let δl = xlindep − xljoint be the hidden-state difference at layer l between independent-segment and joint encoding. The attention error at layer l decomposes into two parts:

  • Propagation term — the part of ΔAttnl caused by δl being fed into Q (and into K, V at layers above l): bounded by C · ‖δl‖ for a Lipschitz constant C, where C is small thanks to QK-Norm (logits ∈ [−τ, τ]).
  • Injection term — the part caused by attending only to in-segment keys/values rather than all keys/values: a fresh, δl-independent error εl with ‖εl‖ ≤ E for some E that QK-Norm again bounds (since both attention outputs are convex combinations of V vectors with bounded weights).
Bound: parallel residual avoids FFN amplification of attention error

In a sequential block, the FFN at layer l sees the corrupted post-attention state, so

δl+1seq = δl + ΔAttnl + &bigl[FFN(δl + ΔAttnl) − FFN(0)&bigr]

and the FFN’s Lipschitz constant LF multiplies the attention error too. Per-layer amplification is roughly (1 + C)(1 + LF), which compounds over depth.

In a parallel block, the FFN sees the same RMSNorm(xl) as attention and is unaffected by ΔAttnl:

δl+1par = δl + αl · ΔAttnl + βl · ΔFFNl

where ΔFFNl depends only on δl (FFN’s sensitivity to its own input), bounded by LF · ‖δl‖. Substituting the attention decomposition gives the recurrence

‖δl+1par‖ ≤ (1 + αC + βLF) · ‖δl‖ + α · E.

Let ρ = 1 + αC + βLF. Solving the linear recurrence with δ0 = 0 (the input embeddings are identical for independent and joint encoding) gives

‖δLpar‖ ≤ α · E · (ρL − 1) / (ρ − 1).

This is bounded growth, not decay: the divergence accumulates with depth, but the per-layer factor ρ is much smaller than its sequential counterpart (1 + C)(1 + LF) because α, β < 1 weight the contributions and decouple attention error from FFN. With α ≈ β ≈ 0.3, C ≈ 1, LF ≈ 1 we get ρpar ≈ 1.6 vs. ρseq ≈ 4.0 per layer.

The micro-correction gate (§4.4) then closes the residual gap. Empirical validation of these Lipschitz constants on a trained model is required; the bound above is an architectural argument, not a quality guarantee. ■

4.4 Component 4: Micro-Correction Gates

Components 1–3 already make the dominant error terms negligible. For any small residual divergence, we store lightweight statistics alongside each segment’s KV cache and apply a correction gate at stitching time.

Cached per-segment statistics

Per segment S, per layer l, per head h, store two vectors:

μs(l,h) = (1/|S|) ∑i∈S ki(l,h) ∈ ℝdhead   (mean of key vectors)
σs(l,h) = stdi∈S(ki(l,h)) ∈ ℝdhead   (standard deviation of key vectors)

Storage overhead for a 32-layer, 32-head, dhead=128 model:
2 vectors × 32 layers × 32 heads × 128 dims × 2 bytes (bf16) = ~0.5 MB per segment. Negligible vs. the KV cache itself (typically 50–200 MB per segment).

Two gate options

We offer a scalar gate (simpler, fewer params) and a vector gate (more expressive) depending on the accuracy/simplicity tradeoff desired:

Scalar gate implementation (Option A, recommended default)

def scalar_gate(seg_stats_pairs, w_gate, b_gate):
    """
    seg_stats_pairs: (batch, n_heads, 2*d_head)
        concatenation of (mu_i - mu_j) and log(sigma_i / sigma_j)
        averaged over all adjacent segment pairs
    w_gate: (n_heads, 2*d_head) per layer
    b_gate: (n_heads,) per layer
    Returns: (batch, n_heads, 1, 1) broadcast-ready gate
    """
    # Contract over feature dim -> one scalar per head
    logit = torch.einsum('bhf,hf->bh', seg_stats_pairs, w_gate) + b_gate
    gate = torch.sigmoid(logit)                    # (batch, n_heads)
    return gate.unsqueeze(-1).unsqueeze(-1)        # (batch, n_heads, 1, 1)

Vector gate implementation (Option B, higher fidelity)

def vector_gate(seg_stats_pairs, W_gate, b_gate):
    """
    seg_stats_pairs: (batch, n_heads, 2*d_head)
    W_gate: (n_heads, d_head, 2*d_head) per layer
    b_gate: (n_heads, d_head) per layer
    Returns: (batch, n_heads, 1, d_head) broadcast-ready gate
    """
    logit = torch.einsum('bhf,hdf->bhd', seg_stats_pairs, W_gate) + b_gate
    gate = torch.sigmoid(logit)                    # (batch, n_heads, d_head)
    return gate.unsqueeze(2)                       # (batch, n_heads, 1, d_head)

# Usage: attn_out = attn_out * vector_gate(seg_stats, self.W_gate, self.b_gate)

Segment statistics preprocessing (computed once per layer at stitch time; the per-layer cached statistics are stacked, then reduced across adjacent segment pairs):

def compute_seg_stats_pairs(seg_stats_list):
    """
    seg_stats_list: list of m (mu, sigma) tuples for ONE layer,
                    each tensor shaped (n_heads, d_head).
                    mu, sigma are computed over BASE (pre-RoPE) keys, so
                    they are position-independent and reusable.
    Returns: (n_heads, 2*d_head) -- averaged pairwise features for this layer.
    """
    deltas = []
    for i in range(len(seg_stats_list) - 1):
        mu_i, sigma_i = seg_stats_list[i]
        mu_j, sigma_j = seg_stats_list[i + 1]
        delta_mu = mu_i - mu_j                                 # (n_heads, d_head)
        # log-ratio rather than ratio: bounded as either sigma -> 0,
        # symmetric around 0, and stable in bf16.
        log_sigma_ratio = torch.log((sigma_i + 1e-6) / (sigma_j + 1e-6))
        deltas.append(torch.cat([delta_mu, log_sigma_ratio], dim=-1))
    return torch.stack(deltas).mean(dim=0)                     # (n_heads, 2*d_head)

This is called once per layer; the per-layer outputs are passed into each HRoPEBlock.forward as the seg_stats argument (see §5).

4.5 Why This Eliminates LayerNorm Leakage

The v3 spec identified “LayerNorm leakage” as the drift in normalization statistics when stitching independently encoded segments. The v5.2 architecture eliminates this at four levels:

Normalization RMSNorm in pre-norm position (before Q/K/V projection) K, V are computed from RMSNormed hidden states per segment. Independent and joint encoding will produce slightly different RMSNorm outputs (different cross-segment context shifts the hidden state), but no post-cache normalization step is needed at stitch time — the normalization happens inside each segment’s forward pass and is amortized.
Logit bounding QK-Norm constrains scores to [−τ, +τ] Hidden state magnitude differences (from missing cross-segment context) do not affect attention scores. Only direction matters, and direction is robust.
Error isolation Parallel residual decouples FFN from attention The FFN branch is unaffected by attention errors. Per-layer amplification reduced from ~(1+C)(1+LF) to 1 + αC + βLF.
Residual correction Cached segment statistics + micro-correction gate Any remaining divergence corrected by a small learned gate (~260K or ~33M params total).
🎯 Net result The 50M-parameter correction model from v3 is eliminated. The architecture makes stitching exact at the attention level and bounds hidden-state divergence to negligible levels through the layers, with a tiny gate to handle residuals. Training is single-stage end-to-end.

5. Complete HRoPE Transformer Layer

class HRoPEBlock(nn.Module):
    def __init__(self, d_model, n_heads, d_head, d_ff, gate_mode='scalar'):
        super().__init__()
        self.n_heads = n_heads
        self.d_head = d_head
        self.gate_mode = gate_mode

        self.norm = RMSNorm(d_model)
        self.W_q = nn.Linear(d_model, n_heads * d_head, bias=False)
        self.W_k = nn.Linear(d_model, n_heads * d_head, bias=False)
        self.W_v = nn.Linear(d_model, n_heads * d_head, bias=False)
        self.W_o = nn.Linear(n_heads * d_head, d_model, bias=False)
        self.tau = nn.Parameter(torch.ones(n_heads) * math.sqrt(d_head))
        self.ffn = SwiGLU(d_model, d_ff)
        self.alpha = nn.Parameter(torch.tensor(0.5))
        self.beta  = nn.Parameter(torch.tensor(0.5))

        # Micro-correction gate
        gate_in = 2 * d_head  # features: delta_mu ++ sigma_ratio
        if gate_mode == 'scalar':
            self.w_gate = nn.Parameter(torch.zeros(n_heads, gate_in))
            self.b_gate = nn.Parameter(torch.zeros(n_heads))
        else:  # 'vector'
            self.W_gate = nn.Parameter(torch.zeros(n_heads, d_head, gate_in))
            self.b_gate = nn.Parameter(torch.zeros(n_heads, d_head))

    def forward(self, x, rope_cos, rope_sin, causal_mask, seg_stats=None):
        B, S, _ = x.shape
        H, D = self.n_heads, self.d_head
        h = self.norm(x)

        Q = self.W_q(h).view(B, S, H, D).transpose(1, 2)  # (B, H, S, D)
        K = self.W_k(h).view(B, S, H, D).transpose(1, 2)
        V = self.W_v(h).view(B, S, H, D).transpose(1, 2)

        # Apply RoPE (hierarchical positions)
        Q = apply_rope(Q, rope_cos, rope_sin)
        K = apply_rope(K, rope_cos, rope_sin)

        # QK-Norm: project onto unit sphere, then scale by learned temperature
        Q = Q / (Q.norm(dim=-1, keepdim=True) + 1e-6)
        K = K / (K.norm(dim=-1, keepdim=True) + 1e-6)

        # ReLU-squared attention with denominator tracking
        scores = torch.einsum('bhsd,bhtd->bhst', Q, K)      # (B, H, S, S)
        scores = scores * self.tau.view(1, H, 1, 1)         # scale by per-head temperature
        # causal_mask: 0 for allowed positions, large negative (e.g. -1e4) for masked.
        # We avoid -inf because clamp(min=0) of -inf is fine but downstream fused
        # ops in bf16 can produce NaN; -1e4 is large enough that ReLU^2 zeros it out.
        scores = scores + causal_mask                       # causal mask + ALiBi bias
        weights = scores.clamp(min=0).square()              # phi(z) = ReLU(z)^2

        denom = weights.sum(dim=-1, keepdim=True) + 1e-6    # (B, H, S, 1)
        attn_out = (weights @ V) / denom                    # (B, H, S, D)

        # Micro-correction gate (only when stitching cached segments).
        # seg_stats is precomputed once at stitch time (see Section 4.4) with shape
        # (H, 2*D); we broadcast it across the batch here.
        if seg_stats is not None:
            stats = seg_stats.unsqueeze(0).expand(B, -1, -1)        # (B, H, 2*D)
            if self.gate_mode == 'scalar':
                logit = torch.einsum('bhf,hf->bh', stats, self.w_gate) + self.b_gate
                gate = torch.sigmoid(logit).unsqueeze(-1).unsqueeze(-1)  # (B,H,1,1)
            else:
                logit = torch.einsum('bhf,hdf->bhd', stats, self.W_gate) + self.b_gate
                gate = torch.sigmoid(logit).unsqueeze(2)                 # (B,H,1,D)
            attn_out = attn_out * gate

        # Project back and parallel residual.
        # Note: this implementation shares one RMSNorm `h` between attention and FFN.
        # PaLM/Gemma use separate norms (norm_attn(x) and norm_ffn(x)); both are valid
        # parallel-residual variants. Sharing keeps parameter count down.
        attn_out = self.W_o(attn_out.transpose(1, 2).reshape(B, S, -1))
        ffn_out = self.ffn(h)   # parallel: SAME input h as attention

        return x + self.alpha * attn_out + self.beta * ffn_out

6. KV Cache Stitching Protocol

At inference time, stitching m pre-encoded segments proceeds as:

Phase A: Prepare cached KV (per segment, one-time) 
Storage convention:
  - K_base_s : pre-RoPE, pre-QK-Norm key projections (W_k * h)
  - V_s      : value projections (W_v * h)
  - mu_s, sigma_s : statistics over K_base_s (i.e., POSITION-INDEPENDENT,
                   so they are reusable when the segment is moved)

For each segment s = 1..m:
    1. Load (K_base_s, V_s, mu_s, sigma_s) from storage
    2. Re-rotate keys to their global positions:
         K_rot_s = R((p_s_global) * theta) * K_base_s              # exact (Section 2.3)
       (If K is cached already rotated to a local position p_s_local, use
         K_rot_s = R((p_s_global - p_s_local) * theta) * K_cached_s
       which is algebraically identical.)
    3. QK-normalize keys:
         K_s = K_rot_s / ||K_rot_s||_2 = K_rot_s / ||K_base_s||_2  # rotation preserves norm
    4. Concatenate: K_full = [K_1 ; K_2 ; ... ; K_m]  along seq dim
                    V_full = [V_1 ; V_2 ; ... ; V_m]

  Compute seg_stats_pairs from {(mu_s, sigma_s)} (see Section 4.4)
Phase B: Query-time attention (per new/edited token) 
For each query token q:
    5. Project & normalize: Q = QK_Norm(RoPE(W_q(RMSNorm(x_q))))
    6. For each segment s, compute partial sums:
         N_s = sum_i phi(tau * Q . K_s[i]) * V_s[i]     # numerator
         D_s = sum_i phi(tau * Q . K_s[i])               # denominator
    7. Combine exactly:
         attn_out = (sum_s N_s) / (sum_s D_s + epsilon)  # EXACT decomposition
    8. Apply micro-correction gate from seg_stats_pairs
    9. Parallel residual:
         x = x + alpha * W_o(attn_out) + beta * FFN(RMSNorm(x))
✅ Exactness guarantee (scope) Step 7 produces the same output as computing attention over the full concatenated sequence given the same query q and the same K, V. In practice the query at layer l and the K, V at layer l > 1 differ slightly between independent and joint encoding due to hidden-state divergence in earlier layers. That divergence is bounded by the per-layer factor ρ = 1 + αC + βLF derived in Section 4.3 (architectural bound, not a quality guarantee — empirical validation required). The micro-correction gate (Step 8) absorbs the residual gap.

7. Mathematical Formulation: Two-Level Hierarchy

7.1 Hierarchical Position Encoding

For a token at sentence index i and token offset j within that sentence, the RoPE angle for dimension pair k is:

Θk(i, j) = αsent,k · i + αtoken,k · j

where each α is a per-dimension frequency vector ∈ ℝd/2:

  • αtoken,k = 10000−2k/d — standard RoPE frequencies (token-level, fast rotation)
  • αsent,k = λ · 10000−2k/d — sentence-level frequencies (slow rotation)

The sentence gap λ is a learned scalar, initialized to the average sentence length in the training corpus (typically 20–40 tokens). This ensures that adjacent sentences are separated by a rotation gap comparable to one sentence’s worth of token positions.

The attention score between tokens at (i, j) and (i′, j′) depends on the difference of their angles:

ΔΘk = αsent,k(i − i′) + αtoken,k(j − j′)

This is a direct consequence of RoPE’s composability (Section 2.2, Property 1).

7.2 Variable-Length Sentences

Document = [S1, S2, …, SN],   Si = [ti,1, …, ti,Li]
  • N — number of sentences (variable per document)
  • Li — number of tokens in sentence i (variable per sentence)

No fixed sentence length is required. The hierarchical encoding adapts to the natural segmentation of the text.

7.3 ALiBi for Cross-Segment Bias (Global Order)

To encode global ordering across segments (beyond what RoPE captures through rotation), we add an ALiBi-style linear bias to the attention scores:

scorefinal = τ · q̂Tk̂ − mh · |segi − segj|

where mh = 2−8h/H is a per-head slope following ALiBi’s geometric progression. This penalizes attention between distant segments, encouraging local coherence while still allowing long-range dependencies through the slower-decaying heads.

📚 Literature Reference The 2025 BiPE (Bilevel Positional Encoding) paper validates mixing local RoPE with global ALiBi, achieving 97% accuracy on arithmetic tasks vs. <70% baseline.

8. The Re-Rotation Kernel (Triton)

# Layout: INTERLEAVED RoPE pairs, i.e. K is laid out per token as
#   [x_0, y_0, x_1, y_1, ..., x_{d/2-1}, y_{d/2-1}].
# This differs from the HALVED layout used by Llama / HuggingFace
#   ([x_0..x_{d/2-1}, y_0..y_{d/2-1}]).
# The PyTorch reference (rope_utils.py) and the calling code MUST use the same
# convention as this kernel; a converter is provided for Llama/Qwen weights.
#
# Launch grid: (seq_len, d_head // 2)   -- one program per (token, RoPE-pair).
@triton.jit
def rerotate_kv_kernel(K_base_ptr, K_out_ptr, start_pos, seq_len, d_head):
    token_idx = tl.program_id(0)             # 0 .. seq_len - 1
    pair_idx  = tl.program_id(1)             # 0 .. d_head/2 - 1  (indexes pairs, NOT dims)

    # Standard RoPE frequencies: theta_i = 10000^(-2i/d) for pair index i.
    theta = 10000.0 ** (-2.0 * pair_idx / d_head)
    angle = (start_pos + token_idx) * theta  # per-token global position

    cos_theta = tl.cos(angle)
    sin_theta = tl.sin(angle)

    offset = token_idx * d_head + pair_idx * 2
    x = tl.load(K_base_ptr + offset)
    y = tl.load(K_base_ptr + offset + 1)

    tl.store(K_out_ptr + offset,     x * cos_theta - y * sin_theta)
    tl.store(K_out_ptr + offset + 1, x * sin_theta + y * cos_theta)
📝 Implementation notes
  • The v3 kernel had a bug: angle = start_pos * theta (same angle for all tokens in the chunk). Fixed in v4+: angle = (start_pos + token_idx) * theta gives each token its own global angle. The PyTorch reference (rope_utils.py) was always correct.
  • Layout consistency is mandatory. This kernel uses the interleaved layout. When integrating with Llama/Qwen weights (halved layout), the K projection output must be permuted into interleaved form before this kernel runs — or, equivalently, write a halved-layout variant of the kernel that loads K[token_idx*d_head + pair_idx] and K[token_idx*d_head + pair_idx + d_head/2] as the (x, y) pair.

9. LSH-Based Fuzzy Cache (SimHash)

def simhash(sentence, hash_bits=64):
    features = sentence.lower().split()
    v = [0] * hash_bits
    for f in features:
        h = hashlib.md5(f.encode()).digest()
        h_int = int.from_bytes(h, 'big')
        for i in range(hash_bits):
            if (h_int >> i) & 1:
                v[i] += 1
            else:
                v[i] -= 1
    signature = 0
    for i in range(hash_bits):
        if v[i] > 0:
            signature |= (1 << i)
    return signature

# "The cat sat" -> 0x7f3a2b1c...
# "The dog sat" -> 0x7f3a2b1d...  (Hamming distance = 1-2 bits)

Cache lookup: compute SimHash of the edited sentence, query the cache for entries with Hamming distance ≤ 3 bits. If found, reuse the cached KV and re-rotate to the new position. If not, encode the sentence from scratch and store the result.

10.1 Fine-Tuning Stages (Path 1)

Step counts below are order-of-magnitude targets for a 3–7B base model. Swapping the attention nonlinearity is a substantial perturbation and recovery to baseline perplexity may require significantly more steps; treat Phase 1 as continued pretraining, not a short fine-tune.

  1. Phase 1 (~20K–50K steps): Replace softmax with ReLU² + QK-Norm. Train with full-context encoding (standard causal attention) to adapt the model to the new attention function. Monitor: perplexity within 0.1 nats of the softmax baseline before proceeding.
  2. Phase 2 (~10K steps): Convert sequential residual to parallel. Introduce segment-independent encoding: randomly split documents into segments, encode independently, stitch KV caches. Train on stitched outputs to learn α and β.
  3. Phase 3 (~5K–10K steps): Enable micro-correction gates. Train with mixed stitched/full-context examples (50/50). Gradually increase sequence length per the schedule above.
