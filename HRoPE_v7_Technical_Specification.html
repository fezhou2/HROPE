HRoPE v7 — Symmetric U-Net for Editable LLMs

A faithful nnU-Net architecture for hierarchical text encoding: encoder ↔ decoder symmetry, skip connections at every level, no tricks.

📄 Feng Zhou — April 2026 Patent Pending Technical Specification v7.1
⚡ What changed from v6 to v7

v6 used the nnU-Net encoder structure (token → sentence → paragraph) but had only a single broadcast step at the very end of the network. That is half a U — an asymmetric design where the decoder path collapses into one cross-attention layer.

v7 completes the U: a full symmetric decoder path with matching transformer stacks at each level (L2_dec, L1_dec, L0_dec) and additive skip connections from every encoder level to its matching decoder level. This is the actual nnU-Net pattern applied to text.

No new mechanisms. No distillation. No auxiliary chunking. No mixture-of-experts. Same transformer block, same RoPE bands, same pools as v6 — just arranged into a complete U.

⚡ v7.0 → v7.1 (this revision)

Added the v4 cache-stitching robustness toolkit as opt-in features in HRoPEBlock:

  • Parallel residual (v4 §4.3) — default ON at L1/L2 to bound error propagation when sentence summaries shift after an edit. Adds 2 scalar params per affected block (8 params total in the default config).
  • Micro-correction gates (v4 §4.4) — optional, default OFF. Enables compensation when a future v8 editor switches L1 from exact re-pass to approximate cache stitching.
  • compute_seg_stats helper that produces the per-segment (μ, log-σ) features the gate consumes.

The reference is gated and zero-initialised so toggling the new flags doesn't change behaviour at init — the smoke test verifies y0 diff = 0.0 between gate-on and gate-off models with shared seeds. See §9 for the full status of v4’s four-component architecture in v7.

1. The Architectural Diff vs v6

v6 (asymmetric)

tokens ↓ L0_enc ──────────┐ skip ↓ pool │ L1_enc ──┐ │ ↓ pool │ │ L2_enc │ │ ↓ │ │ L2 → bcast L1 → bcast tokens ↓ head (one cross-attn broadcast at end)

v7 (symmetric, U-Net)

tokens ↓ L0_enc ──skip─────────► L0_dec → head ↓ pool ↑ unpool + skip L1_enc ──skip─────────► L1_dec ↓ pool ↑ unpool + skip L2_enc ──(bottleneck)─► L2_dec (decoder stack at every level)

1.1 Components added in v7

ComponentRoleCost
l2_dec (n_l2_dec blocks) Bottleneck decoder. Operates on the paragraph-summary stream after the encoder finishes. Negligible (paragraph stream is short)
l1_dec (n_l1_dec blocks) Sentence-stream decoder. Receives unpool(l2_dec) + skip_l1; produces final sentence-summary representations. ~5−15 ms per 100k-token doc
l0_dec (n_l0_dec blocks) Token-stream decoder. Receives unpool(l1_dec) + skip_l0; produces the final per-token representation that feeds the LM head. Still uses intra-sentence-only attention masks. Same as l0_enc (intra-sentence)
unpool() Indexed broadcast: unpool(parent_seq, child_to_parent)[c] = parent_seq[child_to_parent[c]]. Pure gather, zero parameters — the text equivalent of bilinear upsampling in image U-Nets. Free (gather)

1.2 Components removed

  • CrossAttnBlock broadcast_l1, CrossAttnBlock broadcast_l2 — replaced by additive skips. The cross-attention pattern was forcing v6 to use sparse window masks (token attends to {s−1, s, s+1} sentence summaries); v7’s skip is dense, parameter-free, and one operation cheaper.
Parameter parity with v6

To compare v7 against v6 fairly, halve the encoder depths. Default v7 config uses (n_l0_enc=2, n_l0_dec=2, n_l1_enc=1, n_l1_dec=1, n_l2_enc=1, n_l2_dec=1) which has the same parameter count as v6’s (n_layers_l0=4, n_layers_l1=2, n_layers_l2=1) plus a small win from removing the cross-attention blocks.

2. Forward Pass — the Whole Architecture in 30 Lines

def forward(self, doc):
    # ─── ENCODE (going up the hierarchy) ───────────────────────────
    x0 = self.tok_emb(doc.token_ids)
    for blk in self.l0_enc:
        x0 = blk(x0, doc.token_in_sent, attn_mask=intra_sent_mask)
    skip_l0 = x0                              # save for L0 decoder

    x1 = self.sent_pool(x0, doc.sent_id, N_s)
    for blk in self.l1_enc:
        x1 = blk(x1, doc.sent_pos_in_para, attn_mask=causal_l1)
    skip_l1 = x1                              # save for L1 decoder

    x2 = self.para_pool(x1, doc.para_id_per_sent, N_p)
    for blk in self.l2_enc:
        x2 = blk(x2, doc.para_pos, attn_mask=causal_l2)

    # ─── DECODE (coming back down with skips) ──────────────────────
    y2 = x2                                   # bottleneck = top of U
    for blk in self.l2_dec:
        y2 = blk(y2, doc.para_pos, attn_mask=causal_l2)

    y1 = unpool(y2, doc.para_id_per_sent) + skip_l1     # SKIP
    for blk in self.l1_dec:
        y1 = blk(y1, doc.sent_pos_in_para, attn_mask=causal_l1)

    y0 = unpool(y1, doc.sent_id) + skip_l0              # SKIP
    for blk in self.l0_dec:
        y0 = blk(y0, doc.token_in_sent, attn_mask=intra_sent_mask)

    return self.lm_head(self.head_norm(y0))

3. Why a Symmetric U Closes the Gap to Dense

The intuition matches the original nnU-Net result. In a flat dense transformer, every token sees every other token at every layer; the model can route information freely. In v6, every token had to wait until the very last layer to see document context, and that context came through a single sparse cross-attention pass — one bottleneck.

v7 does two things differently:

  1. Iterated mixing in the decoder. The L1 decoder stack runs after the L1 encoder. Sentence summaries are first contextualized going up (encoder) and then refined further going down (decoder, which has access to the bottleneck’s aggregated paragraph context). Each level’s decoder sees a more globally aware input than its encoder did.
  2. Skips preserve high-frequency detail. The L0 decoder receives skip_l0 + unpool(L1_dec). The skip carries the uncompressed token-level representation (sentence-local content). The unpool brings document context. The LM head therefore reads a representation that has both the precision of L0_enc and the global awareness that L2 brought in — without ever putting document-global positions into a per-token vector.
The classic U-Net argument, ported to text

In image segmentation, a deep encoder learns global structure but loses spatial precision (small objects vanish into pooled features). The U-Net solves this by injecting the encoder’s pre-pool features back into the decoder via skip connections, restoring fine-grained detail.

v6 had the same pathology: the only path to the LM head was through the sentence pool, which throws away ~99% of the per-token information (~256 token vectors compressed into 1). The skip connection in v7 restores all of that information at the LM head, while still letting the document context arrive via the broadcast path. The two paths combine additively, exactly as in nnU-Net.

4. Sentence Isolation Invariant — Preserved

The L0 encoder is unchanged from v6: intra-sentence, causal, RoPE positions reset per sentence. So skip_l0[s] still depends only on the tokens of sentence s.

The L0 decoder also uses intra-sentence-only attention. Its input for token t in sentence s is:

y0[s, t] input = skip_l0[s, t] + unpool(L1_dec)[s, t]

Both terms are functions only of (a) the tokens of sentence s and (b) one summary vector from L1_dec. The decoder mixes these intra-sentence. So the v6 invariant holds:

  • Sentence s’s representation depends only on its tokens + a single summary vector reflecting document context.
  • An edit to sentence k (k ≠ s) cannot change skip_l0[s]; it can only change L1_dec, hence the single summary that broadcasts into s.

5. Incremental Edit Cost

Edit a single sentence k on a 100k-token document (~5,000 sentences, ~250 paragraphs):

StepWhat runsCost
1L0_enc on sentence k onlyO(Lk2) — tens of µs
2Splice skip_l0[k] into cacheO(Lk)
3Re-pool sentence k; re-run L1_encO(Ns·d·n_l1_enc) ≈ few ms
4Re-pool affected paragraph; re-run L2_encO(Np·d) ≈ sub-ms
5Run L2_dec, L1_dec (full)~10 ms (sentence stream)
6L0_dec on k’s tokens (intra-sentence)tens of µs
7LM head on k’s tokensnegligible
Total~10−25 ms vs ~5,000 ms full recompute

The L0_dec for non-edited sentences k+1 … Ns−1 is technically also affected (their L1_dec broadcast vector changed because L1 attention is causal). For the standard editor flow — where you only need correct logits at the cursor — you skip those. For a full re-decode of the document tail, add ~5−10 ms more.

6. Training Recipe

Same staged curriculum as v6, with one small change: in Stage 0 we train the L0 encoder + L0 decoder only, with the upper hierarchy bypassed (skip_l1 = 0). This is implemented as stage0_forward() in hrope_v7_train.py:

def stage0_forward(model, doc):
    x0 = model.tok_emb(doc.token_ids)
    for blk in model.l0_enc: x0 = blk(x0, doc.token_in_sent, mask=intra)
    skip_l0 = x0
    y0 = skip_l0                              # bypass: unpool(L1_dec) = 0
    for blk in model.l0_dec: y0 = blk(y0, doc.token_in_sent, mask=intra)
    return model.lm_head(model.head_norm(y0))

Stage 0 trains a complete L0 enc/dec pair on intra-sentence next-token loss. Stages 1–3 unfreeze upward, identical to v6.

9. Cache-Stitching Exactness & the v4 Four-Component Carry-over

This section addresses a recurring question: "Does v7 have the LayerNorm-leakage and correction-model machinery from the v3/v4 specs? If not, doesn’t the architecture lose accuracy after the first edit?"

The short answer: v7’s native architecture sidesteps most of the v3/v4 stitching pathologies by construction, and v7.1 adds the remaining v4 hardening (parallel residual + optional gates) as opt-in features. The detailed accounting follows.

9.1 Why the v3 LayerNorm-leakage scenario doesn’t apply to v7’s native path

The v3/v4 leakage concern was framed for Path 1 retrofit: take an existing Llama-3 / Qwen model trained with full softmax attention and try to swap in cache stitching at inference. In that setting, hidden-state distributions drift away from what the model expects, and a correction model (or micro-gates) is needed to compensate.

v7 is Path 2 native training. Three structural facts make leakage impossible at the L0 layer where it would be most damaging:

  1. RMSNorm, not LayerNorm. RMSNorm has no mean-subtraction step. It normalises each per-token vector by its own L2 magnitude only. There is no cross-token statistic involved — so an edit at sentence k cannot shift sentence j’s normalised representation through a shared mean.
  2. Sentence-isolation enforced by attention mask. The L0 encoder’s attention mask is block-diagonal at the sentence boundary. Sentence k’s tokens never attend to sentence j’s tokens. An edit at k cannot enter j’s key/query/value computation through any attention path.
  3. The model is trained with this exact masking. There is no train/inference distribution mismatch. The hidden states the L0 encoder produces under stitched inputs are the same hidden states it would produce under a fresh full-document forward, because the only thing the L0 encoder ever sees during training is sentence-isolated context.

The smoke test verifies this empirically: after editing sentence 2, skip_l0 at every other sentence is unchanged with max diff 0.0e+00 in fp32.

9.2 Where stitching effects DO appear in v7 — and how they are handled

WhereEffectv7 handling
L1 self-attention (sentence stream) An edit changes sentence k’s pooled summary, which causally affects all sentence positions > k in L1. Exact re-pass. v7’s editor recomputes L1_enc and L1_dec over the spliced sentence stream. There is no “stitch” at L1 — we don’t cache L1’s KV; we cache L1’s input (the sentence summaries) and re-run attention. Cost: ~10 ms on a 100k-token doc with ~5,000 summaries.
L2 self-attention (paragraph stream) Same causal propagation, but at paragraph granularity (~250 paragraphs). Same: exact re-pass. Cost is sub-millisecond.
L0 decoder broadcast input The unpooled L1_dec vector that broadcasts into sentence j’s L0 decoder is different after the edit (L1_dec changed). Sentence j’s L0 decoder output therefore differs. This is correct behaviour, not leakage — it’s the document context updating. The L0 decoder still uses intra-sentence-only attention, and Stage 3 of the curriculum trains the model to handle exactly this kind of shift in the broadcast vector. v7.1 also enables parallel residual at L0_dec (off by default, controlled by parallel_residual_l0) for users who want extra error isolation here.

9.3 The v4 four-component architecture in v7

v4’s four-component architecture (denominator-tracked ReLU², QK-Norm, parallel residual, micro-correction gates) was designed primarily for the Path 1 retrofit. v7.1’s status:

Componentv4 purposev7.1 statusDefault
1. Denominator-tracked ReLU² attention Make per-row attention denominators stitch-decomposable ✅ Implemented (relu_sq_attention returns denom; per-row, exposable for stitching) ON
2. QK-Norm Bound attention logits independent of hidden-state magnitude drift ✅ Implemented (q_norm, k_norm in HRoPEBlock) ON
3. Parallel residual blocks Decouple FFN from attention so error in attention doesn’t pollute FFN; bounds cross-layer error to O(αL) where α ≈ 0.5 ✅ Implemented (parallel_residual=True). Adds 2 scalar parameters per block. ON for L1/L2 (where stitch effects can compound across the decoder); OFF for L0 (sentence-isolated, nothing to compound)
4. Cached segment statistics + micro-correction gates Per-layer sigmoid gate driven by cached (μ, log-σ) features; compensates residual drift after approximate stitching ✅ Implemented (use_micro_gate=True, plus the compute_seg_stats helper). Zero-initialised so the gate is identity at init. OFF (v7’s editor does exact L1 re-pass, so there is no residual drift to gate; turn ON for the v8 incremental-L1 path)

9.5 What is still on the roadmap

  • Bit-exact comparison: stitched edit vs full re-encode. The structural argument in §9.1–9.2 says the two should be byte-identical at L0 and identical-up-to-causal-propagation at L1/L2/L0_dec. A wider parameterised test that runs full forward, edits, runs again, compares against fresh full forward will be added once a real (non-toy) checkpoint is available.
  • Stage 4 stitch fine-tune. Train Stage 3 with the gates active and (μ, log-σ) statistics flowing through, so that the model learns to USE the gate when an approximate L1 update is performed (a v8 prerequisite).
  • v8 incremental-L1 editor. The current editor re-runs L1 self-attention from cached inputs (~10 ms on 5k sentences). A v8 design would do approximate L1 stitching using the gates and segment statistics — targeting sub-millisecond edit cost. v7.1 already ships the parameter machinery this requires.

10. Roadmap from Here

  1. 350M ablation: v6 vs v7. Same data, same parameter count. The hypothesis is that v7’s symmetric U closes 50−70% of the gap to a dense baseline, primarily because of the L0 skip preserving high-frequency detail at the LM head.
  2. If v7 wins by >0.3 PPL, scale to 1.5B with the same staged curriculum (~$60K, ~14 days on 32−64×H100).
  3. Triton kernel for unpool + add fused with first L0_dec attention. The current PyTorch reference does these as two kernel launches; fusing them saves a memory round-trip.
  4. Stage 4 stitch-aware fine-tune. Train with parallel residual + micro-correction gates active and (μ, log-σ) stats flowing through, so the model learns to USE the gate when downstream calls perform approximate L1 stitching.
  5. v8 incremental-L1 editor. Replace the exact L1 re-pass with an approximate stitch using cached segment statistics + the trained gates. Target sub-millisecond edit cost on 100k-token docs.

11. Patent Claims (Refocused)

  1. Claim 1. A hierarchical text encoder-decoder where (i) tokens are encoded by an intra-sentence transformer stack with sentence-resetting RoPE positions, (ii) sentence summaries are encoded by a sentence-level transformer stack with its own RoPE band, (iii) optionally paragraph summaries are encoded by a paragraph-level stack, and (iv) a symmetric decoder path with matching transformer stacks at each level reconstructs token representations using additive skip connections from each encoder level to its matching decoder level.
  2. Claim 2. The use of indexed-gather upsampling (unpool) as the level-to-level transition in the decoder path: each finer-grained element receives its parent’s vector by index lookup.
  3. Claim 3. The use of disjoint rotary frequency bases per hierarchy level, shared between the encoder and decoder of that level.
  4. Claim 4. An incremental edit algorithm that recomputes only the encoder’s skip cache for the edited sentence, the upper-level encoder/decoder passes (which are short), and the decoder pass over the edited sentence’s tokens — preserving all other sentences’ skip caches.
  5. Claim 5. A staged training curriculum where Stage 0 trains the L0 encoder + L0 decoder pair in isolation (upper hierarchy bypassed via zero-summary), and subsequent stages progressively unfreeze the upper levels of the U.
  6. Claim 6. Per-level selective parallel-residual blocks (parallel at the levels that consume cross-segment broadcasts; sequential at the sentence-isolated L0 level), bounding error propagation across decoder layers when upstream summaries shift after an edit.
  7. Claim 7. Sufficient-statistics micro-correction gates driven by per-segment (μ, log-σ) features cached alongside each sentence’s skip representation, enabling approximate cross-segment stitching at the sentence-summary level (L1) with bounded accuracy degradation.
  8. Claim 8. Hierarchical RMSNorm + sentence-isolated attention as a sufficient condition for byte-exact sentence isolation in the L0 encoder cache, eliminating the need for a separate “LayerNorm-leakage correction model” in the native-training path.
